    24→## 2. 模型架构配置
    25→
    26→### 2.1 模型架构矩阵
    27→实验涵盖4种模型架构配置：
    28→
    29→| 模型名称 | Gated Attention | Differential Attention | 估算参数量 |
    30→|---------|-----------------|------------------------|--------|
    31→| Base-0.4B | ❌ | ❌ | ~0.39B |
    32→| GatedAttn-0.4B | ✅ | ❌ | ~0.42B（+gate_proj ~24M） |
    33→| DiffAttn-0.4B | ❌ | ✅ | ~0.42B（+Q翻倍/lambda_proj ~25M） |
    34→| GatedDiffAttn-0.4B | ✅ | ✅ | ~0.44B（+gate_proj+Q翻倍+lambda_proj ~49M） |
    35→
    36→> **注意:** 4 个模型参数量不完全相同。Gated Attention 每层增加 `gate_proj`（d_model × head × head_dim = 1M/层，24层共 ~24M），Differential Attention 每层 Q 投影翻倍 + `lambda_proj`（~1.05M/层，24层共 ~25M）。这是注意力机制本身的固有差异，不影响控制变量实验的有效性——实验目标是研究这些机制对量化敏感性的影响，而非在完全相同参数量下做对比。

   142→### 2.3 注意力机制特殊配置
   143→
   144→#### 2.3.1 Gated Attention（`llm.arch.attention.Attention`）
   145→```python
   146→# 启用后，Attention 模块内部添加 gate_proj 投影层
   147→# 具体实现（attention.py 第68行）：
   148→self.gate_proj = MixPrecisionLinear(args.d_model, self.head * self.head_dim, bias=args.attention_bias)
   149→
   150→# 前向传播中的门控计算（attention.py 第88-91, 125-126行）：
   151→gate = self.gate_proj(x)              # 计算门控信号
   152→output = glu(output, gate)            # glu(x, gate) = x * sigmoid(gate)
   153→```
   154→- **功能:** 为注意力输出添加 sigmoid 门控机制，提升模型表达能力
   155→- **额外参数量:** `d_model × (head × head_dim)` = 1024 × 1024 ≈ 1M 参数/层
   156→
   157→#### 2.3.2 Differential Attention（`llm.arch.attention.Attention`）
   158→```python
   159→# 启用后，Q投影维度翻倍，额外添加 lambda_proj 投影层
   160→# 具体实现（attention.py 第52, 69行）：
   161→q_proj_dim = 2 * self.head * self.head_dim   # Q 维度翻倍
   162→self.lambda_proj = MixPrecisionLinear(args.d_model, self.head, bias=False)
   163→
   164→# 前向传播中的差分计算（attention.py 第121-123行）：
   165→attn1, attn2 = output[:, 0::2], output[:, 1::2]   # 将注意力头分为两组
   166→output = diff(attn1, attn2, self.lambda_proj(x))    # attn1 - sigmoid(λ) * attn2
   167→```
   168→- **功能:** 使用差分注意力计算，将注意力头分为两组并做差分，改善注意力分布
   169→- **额外参数量:** Q 投影翻倍 + lambda_proj ≈ 1M+ 参数/层

   210→
   211→**与项目 `warmup_constant` 的关系：**
   212→- **对齐项目的配置：** `learning_rate=6e-4`, `warmup_iters=2000`, `update_freq=2`, 以及所有优化器超参数（beta1/beta2/weight_decay/clip_grad_norm/adam_eps）
   213→- **实验特有的调整：** `constant_lr=False` + `min_lr=3e-5`（余弦衰减）, `max_updates=10000`, `seed=42`
   214→
   215→**选择余弦衰减而非项目的恒定学习率的理由：**
   216→- 项目的恒定学习率是为**百万步长训练**设计的——训练充分长、随时可停、loss 有足够步数收敛
   217→- 本实验只有 **10K 步**，恒定 `6e-4` 高学习率会导致 loss 尾部持续波动，baseline 自身都未稳定收敛
   218→- 余弦衰减让 loss 曲线尾部更平滑 → baseline vs 量化的 gap 清晰可量化
   219→- baseline 和量化组使用完全相同的衰减策略，差异仍然只来自量化本身
   220→
   221→### 3.2 学习率调度（`llm.warmup_scheduler.WarmupCosineAnnealingLR`）
   222→
   223→使用项目原生实现的 `WarmupCosineAnnealingLR` 调度器，线性预热 + 余弦衰减：
   224→
   225→```python
   226→from llm.warmup_scheduler import WarmupCosineAnnealingLR
   227→
   228→scheduler = WarmupCosineAnnealingLR(
   229→    optimizer=optimizer,
   230→    warmup_steps=2000,      # 预热步数（线性从 0 升至 base_lr）
   231→    T_max=10000,            # 总训练步数
   232→    eta_min=3e-5,           # 最小学习率（峰值的 ~5%）
   233→)
   234→```
   235→
   236→**调度器行为（warmup_scheduler.py 源码逻辑）：**
   237→```
   238→阶段1 - 线性预热 (step 0 → 1999):
   239→  lr = base_lr × (step + 1) / warmup_steps
   240→  即: 0 → 3e-7 → ... → 6e-4（线性增长）
   241→
   242→阶段2 - 余弦衰减 (step 2000 → 9999):
   243→  lr = eta_min + (base_lr - eta_min) × (1 + cos(π × t / T_adjusted)) / 2
   244→  即: 6e-4 → ... → 3e-5（平滑衰减）
   245→```
   246→
   247→**与 nnscaler_train.py 中的集成方式：**
   248→```python
   249→# nnscaler_train.py 第465-473行
   250→lrscheduler_config = LRSchedulerConfig(
   251→    type=WarmupCosineAnnealingLR,
   252→    args={
   253→        'warmup_steps': trainer_args.warmup_iters,  # 2000
   254→        'T_max': trainer_args.max_updates,           # 10000
   255→        'eta_min': trainer_args.min_lr,              # 3e-5
   256→    },
   257→    interval='step',    # 每步更新学习率
   258→)
   259→```

   261→### 3.3 优化器配置（`nnscaler.HybridOptimizer`）
   262→
   263→项目使用 HybridOptimizer，为 embedding 层和其他参数配置不同的权重衰减：
   264→
   265→```python
   266→# nnscaler_train.py 第431-463行
   267→
   268→# 参数分类函数：embedding 层不使用权重衰减
   269→def param_clss_fn(param_name):
   270→    if 'tok_embedding' in param_name:
   271→        return 0, 0    # 类别0：embedding（weight_decay=0）
   272→    else:
   273→        return 1, 0    # 类别1：其他参数（weight_decay=0.1）
   274→
   275→# 主参数优化器配置
   276→main_opt_args = {
   277→    'lr': 6e-4,
   278→    'betas': (0.9, 0.95),
   279→    'weight_decay': 0.1,
   280→    'fused': True,          # 使用 CUDA fused 实现
   281→    'eps': 1e-8,
   282→}
   283→
   284→# Embedding 优化器配置（无权重衰减）
   285→embed_opt_args = {
   286→    'lr': 6e-4,
   287→    'betas': (0.9, 0.95),
   288→    'weight_decay': 0.0,    # embedding 不衰减
   289→    'fused': True,
   290→    'eps': 1e-8,
   291→}
   292→
   293→# 精度选择：fp32 用 AdamW，bf16 用 MixedPrecisionAdamW
   294→sub_opt_type = torch.optim.AdamW  # 或 MixedPrecisionAdamW
   295→
   296→optimizer_config = OptimizerConfig(
   297→    type=nnscaler.HybridOptimizer,
   298→    args={
   299→        'config': {
   300→            'optimizers': [
   301→                {'type': sub_opt_type, 'options': embed_opt_args},
   302→                {'type': sub_opt_type, 'options': main_opt_args},

   306→    param_clss_fn=param_clss_fn,

   340→    'optimizer_states': 'fp32',               # 优化器状态 fp32
   341→    'model_params': 'bf16 (manual cast)',     # 模型参数手动 cast 到 bf16
   342→    'fp32_params': ['gate.weight'],           # MoE gate 权重保持 fp32
   343→    'fp32_buffers': ['cos_sin_cache'],        # RoPE 缓存保持 fp32
   344→    'gradient_allreduce': 'fp32',             # --accumulate_allreduce_grads_in_fp32

   432→- **实验有效性:** 合成随机数据无法反映量化对真实信号传播的影响

   647→**组件:** `llm.kernel.vocab_parallel_cross_entropy.inner_chunk_linear_cross_entropy`
   648→
   649→```python
   650→# BackBone.forward（nnscaler_train.py 第155行）
   651→losses, z_losses = inner_chunk_linear_cross_entropy(
   652→    hidden_states, self.model.output.weight, targets,
   653→    self.xentropy_chunk_size,   # chunk 大小
   654→    self.xentropy_recompute     # 是否重计算以节省内存
   655→)
   656→
   657→# WrapperModel.forward 中的损失聚合：
   658→loss = torch.sum(losses * loss_mask)          # 主语言模型损失
   659→z_loss = torch.sum(z_losses * loss_mask)      # Z-loss
   660→# 最终损失 = lm_loss + z_loss_scale * z_loss
   661→# 注意：本实验中 z_loss_scale=0（默认值），z_loss 不影响训练
   662→```
