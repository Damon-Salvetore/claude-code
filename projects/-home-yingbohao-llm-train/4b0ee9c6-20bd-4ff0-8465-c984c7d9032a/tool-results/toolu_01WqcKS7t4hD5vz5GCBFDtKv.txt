   143→### 2.3 注意力机制特殊配置
   144→
   145→#### 2.3.1 Gated Attention（`llm.arch.attention.Attention`）
   146→```python
   147→# 启用后，Attention 模块内部添加 gate_proj 投影层
   148→# 具体实现（attention.py 第68行）：
   149→self.gate_proj = MixPrecisionLinear(args.d_model, self.head * self.head_dim, bias=args.attention_bias)
   150→
   151→# 前向传播中的门控计算（attention.py 第88-91, 125-126行）：
   152→gate = self.gate_proj(x)              # 计算门控信号
   153→output = glu(output, gate)            # glu(x, gate) = x * sigmoid(gate)
   154→```
   155→- **功能:** 为注意力输出添加 sigmoid 门控机制，提升模型表达能力
   156→- **额外参数量:** `d_model × (head × head_dim)` = 1024 × 1024 ≈ 1M 参数/层
   157→
   158→#### 2.3.2 Differential Attention（`llm.arch.attention.Attention`）
   159→```python
   160→# 启用后，Q投影维度翻倍，额外添加 lambda_proj 投影层
   161→# 具体实现（attention.py 第52, 69行）：
   162→q_proj_dim = 2 * self.head * self.head_dim   # Q 维度翻倍
   163→self.lambda_proj = MixPrecisionLinear(args.d_model, self.head, bias=False)
   164→
   165→# 前向传播中的差分计算（attention.py 第121-123行）：
   166→attn1, attn2 = output[:, 0::2], output[:, 1::2]   # 将注意力头分为两组
   167→output = diff(attn1, attn2, self.lambda_proj(x))    # attn1 - sigmoid(λ) * attn2
   168→```
   169→- **功能:** 使用差分注意力计算，将注意力头分为两组并做差分，改善注意力分布
   170→- **额外参数量:** Q 投影翻倍 + lambda_proj ≈ 1M+ 参数/层

   209→
   210→**与项目 `warmup_constant` 的关系：**
   211→- **对齐项目的配置：** `learning_rate=6e-4`, `warmup_iters=2000`, `update_freq=2`, 以及所有优化器超参数（beta1/beta2/weight_decay/clip_grad_norm/adam_eps）
   212→- **实验特有的调整：** `min_lr=3e-5`（余弦衰减）, `max_updates=10000`, `seed=42`
   213→
   214→**关于余弦衰减的实现细节：**
   215→- `TrainerArgs` 中的 `constant_lr` 和 `lr_decay_iters` 字段在 `nnscaler_train.py` 中**未被使用**
   216→- 调度器 `WarmupCosineAnnealingLR` 只读取 `warmup_iters`、`max_updates`（作为 T_max）和 `min_lr`（作为 eta_min）
   217→- 项目实现"恒定学习率"的方式是设置 `min_lr = learning_rate`（如 `--min_lr 6e-4 --learning_rate 6e-4`），使余弦衰减振幅为零
   218→- 本实验设置 `min_lr=3e-5`（远小于 `learning_rate=6e-4`），因此 scheduler 会执行正常的余弦衰减
   219→
   220→### 3.2 学习率调度（`llm.warmup_scheduler.WarmupCosineAnnealingLR`）
   221→
   222→使用项目原生实现的 `WarmupCosineAnnealingLR` 调度器，线性预热 + 余弦衰减：
   223→
   224→```python
   225→from llm.warmup_scheduler import WarmupCosineAnnealingLR
   226→
   227→scheduler = WarmupCosineAnnealingLR(
   228→    optimizer=optimizer,
   229→    warmup_steps=2000,      # 预热步数（线性从 0 升至 base_lr）
   230→    T_max=10000,            # 总训练步数
   231→    eta_min=3e-5,           # 最小学习率（峰值的 ~5%）
   232→)
   233→```
   234→
   235→**调度器行为（warmup_scheduler.py 源码逻辑）：**
   236→```
   237→阶段1 - 线性预热 (step 0 → 1999):
   238→  lr = base_lr × (step + 1) / warmup_steps
   239→  即: 0 → 3e-7 → ... → 6e-4（线性增长）
   240→
   241→阶段2 - 余弦衰减 (step 2000 → 9999):
   242→  lr = eta_min + (base_lr - eta_min) × (1 + cos(π × t / T_adjusted)) / 2
   243→  即: 6e-4 → ... → 3e-5（平滑衰减）
   244→```
   245→
   246→**与 nnscaler_train.py 中的集成方式：**
   247→```python
   248→# nnscaler_train.py 第465-473行
   249→lrscheduler_config = LRSchedulerConfig(
   250→    type=WarmupCosineAnnealingLR,
   251→    args={
   252→        'warmup_steps': trainer_args.warmup_iters,  # 2000
   253→        'T_max': trainer_args.max_updates,           # 10000
   254→        'eta_min': trainer_args.min_lr,              # 3e-5
   255→    },
   256→    interval='step',    # 每步更新学习率
   257→)
   258→```

   328→### 3.5 精度配置
   329→**精度配置:** `--precision fp32`（项目标准混合精度模式）
   330→
   331→虽然 CLI 参数名为 `fp32`，但实际上是**前向 bf16、梯度 fp32** 的混合精度训练。这是通过模型内部的 `MixPrecisionLinear` 和 `RMSNorm` 实现的，与 `BackBone.__init__` 中的 precision 参数无关。
   332→
   333→```python
   334→# 混合精度的实现机制（模型架构层面，非 BackBone.__init__）：
   335→
   336→# 1. 所有线性层使用 MixPrecisionLinear（arch/linear.py）
   337→#    前向：assert input.dtype == torch.bfloat16 → F.linear(input, weight.to(bf16))
   338→#    反向：grad_weight = torch.mm(grad_output.t(), input, out_dtype=torch.float32)
   339→#    → 前向 bf16 计算，权重梯度回传为 fp32
   340→
   341→# 2. RMSNorm 强制 bf16 计算（arch/rms_norm.py 第20行）
   342→#    F.rms_norm(x.to(torch.bfloat16), ..., weight=weight.to(torch.bfloat16))
   343→
   344→# 3. 残差连接在 fp32 下进行（arch/model.py Block.forward）
   345→#    r = self.self_attn(...)   # bf16 输出
   346→#    h = x + r.float()         # .float() 转 fp32 后做残差加法
   347→#    r = self.mlp(...)         # bf16 输出
   348→#    out = h + r.float()       # 同上
   349→
   350→# 4. tok_embeddings 是标准 nn.Embedding（fp32 权重），输出 .float() 转 fp32
   351→#    h = self.tok_embeddings(tokens).float()  # model.py 第73行
   352→
   353→# 使用范围：ALL linear layers（q/k/v/o_proj, gate_proj, lambda_proj,
   354→#           up/down/gate_proj in FFN, output head）均为 MixPrecisionLinear
   355→# 注意：tok_embeddings 是标准 nn.Embedding，不是 MixPrecisionLinear
   356→```
   357→
   358→```yaml
   359→# --precision fp32 模式（本实验使用）：
   360→precision_config:
   361→  model_weights_storage: "fp32"           # 权重以 fp32 存储
   362→  forward_computation: "bf16"             # 线性层和 RMSNorm 在 bf16 下计算
   363→  residual_stream: "fp32"                # 残差连接在 fp32 下（r.float() 后加法）
   364→  weight_gradients: "fp32"               # 权重梯度 fp32（out_dtype=torch.float32）
   365→  optimizer_type: "torch.optim.AdamW"    # 标准 AdamW
   366→  optimizer_states: "fp32"               # m/v 状态 fp32
   367→
   368→# --precision bf16 模式（本实验不使用）：
   369→#   在上述基础上，BackBone.__init__ 额外将权重存储也 cast 到 bf16
   370→#   优化器改为 MixedPrecisionAdamW
   371→#   自动启用 --accumulate_allreduce_grads_in_fp32
   372→```
   373→
   374→**本质：** `--precision fp32` 是 **"权重 fp32 + 前向 bf16 + 梯度 fp32"** 的混合精度模式。所有 `MixPrecisionLinear` 层在前向时自动将输入/权重转为 bf16 计算，但反向传播中权重梯度以 fp32 精度累积，优化器在 fp32 下更新权重。

   465→## 5. NVFP4量化配置
   466→
   467→> **参考实现:** `/home/yingbohao/Model-Optimizer`（NVIDIA TensorRT Model Optimizer）
   468→> 本节基于 Model-Optimizer 的实际源码分析，描述 NVFP4 伪量化的完整实现细节。
   469→
   470→### 5.1 NVFP4 三级量化体系
   471→
   472→NVFP4 使用三级量化层次结构，实现高效的 4-bit 伪量化：
   473→
   474→```
   475→┌─────────────────────────────────────────────────────────────────┐
   476→│                    NVFP4 三级量化体系                             │
   477→├─────────────────────────────────────────────────────────────────┤
   478→│ Level 3: Global Scale (FP32)                                    │
   479→│   global_scale = global_amax / (6.0 × 448.0)                   │
   480→│   → 单个标量，覆盖整个张量                                       │
   481→│                                                                 │
   482→│ Level 2: Per-Block Scale (FP8 E4M3)                             │
   483→│   block_scale = block_amax / 6.0                                │
   484→│   block_scale_fp8 = FP8_quantize(block_scale / global_scale)    │
   485→│                     × global_scale                              │
   486→│   → 每 16 个元素共享一个 FP8 尺度因子                             │
   487→│                                                                 │
   488→│ Level 1: Values (FP4 E2M1)                                      │
   489→│   normalized = x / block_scale_fp8                              │
   490→│   quantized = round_to_nearest_fp4(normalized)                  │
   491→│   → 每个元素量化到 4-bit E2M1 格式                               │
   492→│                                                                 │
   493→│ Fake Quant Output = quantized × block_scale_fp8                 │
   494→└─────────────────────────────────────────────────────────────────┘
   495→```
   496→
   497→**关键常数:**
   498→- `6.0` — FP4 E2M1 最大可表示值
   499→- `448.0` — FP8 E4M3 最大可表示值
   500→- `block_size = 16` — 每个量化块包含 16 个元素
   501→
   502→### 5.2 FP4 E2M1 值表（16个量化级别）
   503→
   504→```python
   505→# 8个正值 + 8个对应负值 = 16个量化级别
   506→FP4_E2M1_VALUES = [0.0, 0.5, 1.0, 1.5, 2.0, 3.0, 4.0, 6.0]  # 正值
   507→# 对应的量化边界（用于最近值舍入）：
   508→FP4_E2M1_BOUNDS = [0.25, 0.75, 1.25, 1.75, 2.5, 3.5, 5.0]
   509→
   510→# Triton 内核中的量化逻辑（fp4_kernel.py:214-239）：
   511→# abs_scaled = |x| / block_scale
   512→# q_val = nearest_fp4(abs_scaled):
   513→#   abs_scaled ≤ 0.25  → 0.0
   514→#   0.25 < abs_scaled < 0.75  → 0.5
   515→#   0.75 ≤ abs_scaled ≤ 1.25  → 1.0
   516→#   1.25 < abs_scaled < 1.75  → 1.5
   517→#   1.75 ≤ abs_scaled ≤ 2.5   → 2.0
   518→#   2.5  < abs_scaled < 3.5   → 3.0
   519→#   3.5  ≤ abs_scaled ≤ 5.0   → 4.0
   520→#   abs_scaled > 5.0           → 6.0（饱和）
   521→# output = sign(x) × q_val × block_scale
   522→```
   523→
   524→### 5.3 量化范围
   525→
   526→本实验支持两种 NVFP4 量化范围，通过 `nvfp4_mode` 参数控制：
   527→
   528→#### 模式 1: MLP-Only（`nvfp4_mode=1`）
   529→
   530→仅对 FFN（MLP）层做 NVFP4 伪量化，对应 Model-Optimizer 的 `NVFP4_MLP_ONLY_CFG`：
   531→
   532→```python
   533→# Model-Optimizer 中的配置（config.py）：
   534→NVFP4_MLP_ONLY_CFG = {
   535→    "quant_cfg": {
   536→        # 仅匹配 MLP 层内的量化器（fnmatch 模式匹配）
   537→        "*mlp*weight_quantizer": {
   538→            "num_bits": (2, 1),          # E2M1 = FP4 格式
   539→            "block_sizes": {-1: 16, "type": "dynamic", "scale_bits": (4, 3)},
   540→            #               ↑ block=16   ↑ 动态 amax       ↑ E4M3 = FP8 尺度
   541→            "enable": True,
   542→            "pass_through_bwd": True,    # 反向传播使用纯 STE
   543→        },
   544→        "*mlp*input_quantizer": {
   545→            "num_bits": (2, 1),
   546→            "block_sizes": {-1: 16, "type": "dynamic", "scale_bits": (4, 3)},
   547→            "enable": True,
   548→            "pass_through_bwd": True,
   549→        },
   550→        # 所有非 MLP 的量化器保持禁用
   551→        "*lm_head*": {"enable": False},
   552→        "*router*": {"enable": False},
   553→        # ... 其余层全部禁用
   554→    },
   555→    "algorithm": "max",
   556→}
   557→```
   558→
   559→**模式匹配机制:** Model-Optimizer 使用 `fnmatch.fnmatch(module_name, pattern)` 进行层名匹配。
   560→在 llm-train 项目中，FFN 层的命名为：
   561→```
   562→# llm-train 中 FFN 层的 named_modules 路径：
   563→layers.0.mlp.up_proj      → ✅ 匹配 *mlp*（up 投影）
   564→layers.0.mlp.down_proj    → ✅ 匹配 *mlp*（down 投影）
   565→layers.0.mlp.gate_proj    → ✅ 匹配 *mlp*（gate 投影，SwiGLU）
   566→layers.0.self_attn.q_proj → ❌ 不匹配（注意力层，不量化）
   567→layers.0.self_attn.o_proj → ❌ 不匹配（注意力层，不量化）
   568→output                    → ❌ 不匹配（LM head，不量化）
   569→```
   570→
   571→#### 模式 2: FFN + QKVO 全量化（`nvfp4_mode=2`）
   572→
   573→在模式 1 的基础上，额外对注意力层的 Q/K/V/O 投影也做 NVFP4 伪量化：
   574→
   575→```
   576→# nvfp4_mode=2 时的量化层（在模式 1 基础上新增）：
   577→layers.0.self_attn.q_proj → ✅ 量化（Q 投影，权重+输入激活）
   578→layers.0.self_attn.k_proj → ✅ 量化（K 投影，权重+输入激活）
   579→layers.0.self_attn.v_proj → ✅ 量化（V 投影，权重+输入激活）
   580→layers.0.self_attn.o_proj → ✅ 量化（O 投影，权重+输入激活）
   581→layers.0.self_attn.gate_proj → ✅ 量化（Gated Attention 的 gate 投影，仅 GatedAttn 模型）
   582→layers.0.self_attn.lambda_proj → ❌ 不量化（Differential Attention 的 lambda 投影，输出维度太小）
   583→output                    → ❌ 不匹配（LM head，不量化）
   584→```
   585→
   586→**设计说明：**
   587→- `lambda_proj` 不量化：其输出维度仅为 head 数（16），量化意义不大且可能影响 diff attention 的精度
   588→- Q/K/V 共享同一份输入激活的 fake quant（`x_fq = nvfp4_fake_quant(x)`），避免重复量化
   589→- O projection 在 attention 输出（经过 rearrange/gated attention）后做 fake quant
   590→
   591→### 5.4 伪量化前向流程（Fake Quantization Forward）
   592→
   593→在训练的前向传播中，各层的计算流变为：
   594→
   595→#### FFN 层（`nvfp4_mode >= 1`）
   596→
   597→```python
   598→# 原始 FeedForwardNetwork.forward（无量化）：
   599→def forward(self, x):
   600→    return self.down_proj(swiglu(self.up_proj(x), self.gate_proj(x)))
   601→
   602→# 加入 NVFP4 伪量化后的等效流程：
   603→def forward_with_nvfp4(self, x):
   604→    # 1. 对输入激活做 NVFP4 伪量化（W4A4 的 A4 部分）
   605→    x_fq = nvfp4_fake_quant(x)           # 量化+反量化，引入量化噪声
   606→
   607→    # 2. 对权重做 NVFP4 伪量化（W4A4 的 W4 部分）
   608→    up_w_fq = nvfp4_fake_quant(self.up_proj.weight)
   609→    gate_w_fq = nvfp4_fake_quant(self.gate_proj.weight)
   610→
   611→    # 3. 使用伪量化后的权重和激活计算
   612→    up = F.linear(x_fq, up_w_fq)
   613→    gate = F.linear(x_fq, gate_w_fq)
   614→    intermediate = swiglu(up, gate)
   615→
   616→    # 4. 中间激活也做伪量化
   617→    inter_fq = nvfp4_fake_quant(intermediate)
   618→    down_w_fq = nvfp4_fake_quant(self.down_proj.weight)
   619→    return F.linear(inter_fq, down_w_fq)
   620→```
   621→
   622→#### Attention QKVO 层（`nvfp4_mode >= 2`，新增）
   623→
   624→```python
   625→# 原始 Attention.forward 中 QKVO 投影（无量化）：
   626→def forward(self, x, ...):
   627→    xq = self.q_proj(x)       # MixPrecisionLinear
   628→    xk = self.k_proj(x)
   629→    xv = self.v_proj(x)
   630→    # ... attention computation ...
   631→    output = self.o_proj(rearranged_output)
   632→
   633→# nvfp4_mode >= 2 时的等效流程：
   634→def forward_with_nvfp4_qkvo(self, x, ...):
   635→    # 1. Q/K/V 共享同一份输入 fake quant（避免重复量化）
   636→    x_fq = nvfp4_fake_quant(x)
   637→    xq = mix_precision_linear(x_fq, nvfp4_fake_quant(self.q_proj.weight))
   638→    xk = mix_precision_linear(x_fq, nvfp4_fake_quant(self.k_proj.weight))
   639→    xv = mix_precision_linear(x_fq, nvfp4_fake_quant(self.v_proj.weight))
   640→
   641→    # 2. Gated Attention 的 gate_proj 也做量化（如果存在）
   642→    if self.gate_proj is not None:
   643→        gate = mix_precision_linear(nvfp4_fake_quant(x), nvfp4_fake_quant(self.gate_proj.weight))
   644→
   645→    # 3. ... attention computation (RoPE, flash_attn, etc.) 不受影响 ...
   646→
   647→    # 4. O projection 量化
   648→    output = mix_precision_linear(nvfp4_fake_quant(attn_output), nvfp4_fake_quant(self.o_proj.weight))
   649→```
   650→
   651→**注意：** `lambda_proj`（Differential Attention）不做量化，因为其输出维度仅为 head 数（16），太小没有量化收益。
   652→
   653→**量化点对照表（对标 MoE FP8 已有实现）：**
   654→
   655→| 量化点 | MoE FP8（kernel/quant.py 已有） | Dense FFN NVFP4 mode=1 | FFN+QKVO NVFP4 mode=2（新增） |
   656→|--------|-------------------------------|--------------------------|-------------------------------|
   657→| FFN 输入激活 | `act_quant_fp8` before dispatch | `nvfp4_fake_quant(x)` before up/gate | 同 mode=1 |
   658→| W1/W3 权重 | `weight_quant_fp8_(w13)` | `nvfp4_fake_quant(up/gate_proj.weight)` | 同 mode=1 |
   659→| SwiGLU 后中间激活 | `fused_silu` 内嵌量化 | `nvfp4_fake_quant(intermediate)` | 同 mode=1 |
   660→| W2 权重 | `weight_quant_fp8_(w2)` | `nvfp4_fake_quant(down_proj.weight)` | 同 mode=1 |
   661→| Attention 输入激活 | — | — | `nvfp4_fake_quant(x)` before Q/K/V |
   662→| Q/K/V 权重 | — | — | `nvfp4_fake_quant(q/k/v_proj.weight)` |
   663→| Attention 输出激活 | — | — | `nvfp4_fake_quant(output)` before O proj |
   664→| O 权重 | — | — | `nvfp4_fake_quant(o_proj.weight)` |
   665→| 反向传播 | STE 梯度直通 | STE 梯度直通 | STE 梯度直通 |
   666→
   667→### 5.5 梯度处理：直通估计器（STE）
   668→
   669→**反向传播使用纯 STE（`pass_through_bwd=True`）：**
   670→
   671→```python
   672→# Model-Optimizer 的 STE 实现（tensor_quant.py:286-311）
   673→
   674→# 方式1: pass_through_bwd=True（本实验使用）
   675→# → 不保存 inputs/amax，梯度完全直通
   676→# → ∂L/∂x = ∂L/∂output（恒等映射）
   677→# → 内存更省（不需保存前向张量）
   678→
   679→class StaticBlockwiseFP4FakeQuantFunction(torch.autograd.Function):
   680→    @staticmethod
   681→    def forward(ctx, x, amax, global_amax, quantize_block_scales, out_dtype, pass_through_bwd):
   682→        # pass_through_bwd=True 时不保存任何张量
   683→        if not pass_through_bwd and amax is not None:
   684→            ctx.save_for_backward(x, amax)
   685→        # 调用 Triton 内核做前向伪量化
   686→        return static_blockwise_fp4_fake_quant(x, amax, global_amax, quantize_block_scales, out_dtype)
   687→
   688→    @staticmethod
   689→    def backward(ctx, grad_outputs):
   690→        if len(ctx.saved_tensors) == 0:
   691→            # pass_through_bwd=True: 梯度完全直通
   692→            return (grad_outputs, None, None, None, None, None)
   693→        else:
   694→            # pass_through_bwd=False: 超出 amax 的梯度置零
   695→            inputs, amax = ctx.saved_tensors
   696→            return (torch.where(inputs.abs() <= amax, grad_outputs, 0.0), None, None, None, None, None)
   697→
   698→# 方式2: pass_through_bwd=False（可选，内存更多但梯度更精确）
   699→# → ∂L/∂x = ∂L/∂output × 𝟙(|x| ≤ amax)
   700→# → 饱和区（|x| > amax）的梯度被置零
   701→# → 需要额外保存 inputs 和 amax 张量
   702→```
   703→
   704→**为什么 MLP-Only 使用 `pass_through_bwd=True`：**
   705→- 动态量化格式下，amax 每次前向都重新计算，几乎不存在截断（clipping），因此置零梯度没有意义
   706→- 纯 STE 节省内存（不需保存前向输入张量）
   707→- Model-Optimizer 官方文档："For dynamic quantization formats like MXFP4, STE with zeroed outlier gradients is not needed since fake quantization with dynamic amax results in minimal/no clipping"
   708→
   709→### 5.6 Train from Scratch vs QAT 的区别
   710→
   711→Model-Optimizer 的 NVFP4 实现面向 **已有预训练模型的 QAT（Quantization-Aware Training）**，而本实验是 **train from scratch**，两者有关键区别：
   712→
   713→```
   714→┌──────────────────────────────────┬──────────────────────────────────┐
   715→│   Model-Optimizer (QAT 微调)      │   本实验 (Train from Scratch)     │
   716→├──────────────────────────────────┼──────────────────────────────────┤
   717→│ 1. 加载预训练 checkpoint          │ 1. 随机初始化权重                  │
   718→│ 2. 插入量化器（模块替换）          │ 2. 模型定义时直接内建伪量化         │
   719→│ 3. 校准（calibration forward）    │ 3. 无需校准，动态 amax 即可        │
   720→│ 4. QAT 微调（少量步数，小 lr）     │ 4. 全程训练（10K步，量化从头开始）   │
   721→│                                  │                                  │
   722→│ 需要先全精度训练/微调（Step 1）     │ 不需要，从 step 0 直接带量化训练    │
   723→│ → NVIDIA 官方："跳过 Step 1 会     │ → 随机权重没有已学到的信息需要保护   │
   724→│   导致精度变差"                   │ → BitNet 实验证明 warmup 无改善     │
   725→│                                  │                                  │
   726→│ 需要 Triton 高性能内核            │ PyTorch 原生实现即可               │
   727→│ 需要 NVFP4StaticQuantizer        │ 简单的 autograd Function          │
   728→│ 需要模块替换 + fnmatch 匹配       │ 直接在 FFN forward 中插入          │
   729→│ 需要校准数据 + 全局 amax 统计     │ 动态 per-block amax，无需预统计    │
   730→└──────────────────────────────────┴──────────────────────────────────┘
   731→```
   732→
   733→**简化要点：**
   734→- **从 step 0 开始量化，不需要预热不量化阶段：** 从头训练的模型权重是随机初始化的，不存在"已学到的信息被量化噪声破坏"的问题。HuggingFace BitNet 实验在 SmolLM 135M 上验证了从头训练时"有量化 warmup"vs"无量化 warmup"loss 曲线几乎一致。NVIDIA Model-Optimizer 推荐的"先全精度再 QAT"仅适用于已有 checkpoint 的微调场景。
   735→- **无需校准步骤：** 动态 amax（每次前向重新计算 per-block 最大值），不需要提前跑 calibration forward
   736→- **无需模块替换框架：** 不需要 Model-Optimizer 的 `replace_quant_module` + `set_quantizer_by_cfg` 管线
   737→- **PyTorch 原生实现：** 用 `torch.compile` + PyTorch 运算即可，不需要 Triton 内核
   738→
   739→### 5.7 在 llm-train 中的集成方案
   740→
   741→需要在 llm-train 项目中新增 NVFP4 伪量化实现，使用纯 PyTorch 实现：
   742→
   743→```python
   744→# === 需新增的代码模块 ===
   745→
   746→# 1. kernel/nvfp4_quant.py — NVFP4 伪量化（纯 PyTorch 实现）
   747→#    参考算法: Model-Optimizer 的 FP4 E2M1 量化逻辑
   748→#    参考风格: kernel/quant.py 中的 weight_quant（torch.compile + autograd Function）
   749→
   750→# 2. arch/config.py — 添加量化模式配置
   751→#    新增字段: nvfp4_mode: int = 0  # 0: 禁用, 1: FFN NVFP4 伪量化, 2: FFN+QKVO 全量化
   752→
   753→# 3. arch/ffn.py — 在 FeedForwardNetwork.forward 中插入伪量化
   754→
   755→# 4. arch/attention.py — 在 Attention.forward 中对 QKVO 投影插入伪量化（nvfp4_mode >= 2）
   756→
   757→# 5. arch/model.py — 将 nvfp4_mode 传递到 FFN 层
   758→```

   760→**关键设计决策：**
   761→- **纯 PyTorch 实现**（`torch.compile` 优化，不依赖 Triton 内核）
   762→- **block_size = 16**（NVFP4 标准，不同于 FP8 的 128）
   763→- **模式 1: 仅量化 FFN**（up_proj, gate_proj, down_proj 的权重和输入激活）
   764→- **模式 2: FFN + QKVO 全量化**（在模式 1 基础上，额外量化 q_proj, k_proj, v_proj, o_proj 及 gated attention 的 gate_proj）
   765→- **不量化 LM head 和 lambda_proj**（前者影响输出精度，后者维度太小无收益）
   766→- **STE 反向传播**（梯度完全直通，不保存前向张量）
   767→- **动态 amax**（每次前向重新计算 per-block amax，无需校准步骤）
   768→- **从第一步开始量化**（train from scratch，无需预热或渐进量化）

   931→# 已有的 FP8 量化组件（kernel/quant.py）：
   932→# - w_fake_fp8 / x_fake_fp8: Triton 内核，block_size=128, scale=max(|block|)/448
   933→# - weight_quant_fp8_(x): 3D [E,M,N] 权重 FP8 伪量化

   939→#   fp8_mode=1: FP8 伪量化（MoE 训练用）
   940→#   fp8_mode=2: FP8 真量化

   942→
   943→**NVFP4 量化（需要新增实现，纯 PyTorch）：**
   944→
   945→NVFP4 伪量化在项目中**尚未实现**，需参考 Model-Optimizer 的算法逻辑，用纯 PyTorch 实现。
   946→
   947→> **与 Model-Optimizer 的关键区别：** Model-Optimizer 面向已有 checkpoint 的 QAT（需要模块替换、校准、Triton 高性能内核）。本实验是 train from scratch，用 PyTorch 原生实现即可，不需要校准步骤，量化从第一步开始。
   948→
   949→```python
   950→# === 新增模块: kernel/nvfp4_quant.py ===
   951→# 参考算法: Model-Optimizer 的 FP4 E2M1 量化逻辑
   952→# 参考风格: kernel/quant.py 中已有的 weight_quant（torch.compile + autograd Function）
   953→
   954→import torch
   955→
   956→# FP4 E2M1 量化值和边界表
   957→FP4_VALUES = torch.tensor([0.0, 0.5, 1.0, 1.5, 2.0, 3.0, 4.0, 6.0])
   958→FP4_BOUNDS = torch.tensor([0.25, 0.75, 1.25, 1.75, 2.5, 3.5, 5.0])
   959→
   960→@torch.compile
   961→def _nvfp4_fake_quant_impl(x: torch.Tensor, block_size: int = 16) -> torch.Tensor:
   962→    """NVFP4 伪量化核心实现（纯 PyTorch，可被 torch.compile 优化）。
   963→
   964→    算法：
   965→      1. 将张量按 block_size 分块
   966→      2. 计算每块的 amax，得到 scale = amax / 6.0
   967→      3. 归一化: abs_scaled = |x| / scale
   968→      4. 映射到最近的 FP4 E2M1 值（searchsorted）
   969→      5. 重建: output = sign(x) × fp4_value × scale
   970→    """
   971→    orig_shape = x.shape
   972→    # 展平为 [num_blocks, block_size]
   973→    x_2d = x.reshape(-1, block_size).float()
   974→
   975→    # Per-block scale: amax / 6.0（FP4 最大可表示值）
   976→    amax = x_2d.abs().amax(dim=-1, keepdim=True)
   977→    scale = amax / 6.0
   978→    scale = scale.clamp(min=1e-12)  # 避免除零
   979→
   980→    # 归一化到 FP4 范围
   981→    abs_scaled = x_2d.abs() / scale
   982→
   983→    # 映射到最近的 FP4 值（searchsorted + 边界表）
   984→    bounds = FP4_BOUNDS.to(x.device)
   985→    values = FP4_VALUES.to(x.device)
   986→    indices = torch.searchsorted(bounds, abs_scaled)
   987→    q_val = values[indices]
   988→
   989→    # 恢复符号并乘以 scale
   990→    result = torch.sign(x_2d) * q_val * scale
   991→
   992→    return result.to(x.dtype).reshape(orig_shape)
   993→
   994→
   995→class NvFP4FakeQuant(torch.autograd.Function):
   996→    """NVFP4 伪量化 autograd Function，STE 反向传播。
   997→
   998→    前向：量化+反量化（引入量化噪声）
   999→    反向：梯度完全直通（不保存前向张量，不裁剪梯度）
  1000→    """
  1001→    @staticmethod
  1002→    def forward(ctx, x: torch.Tensor, block_size: int = 16) -> torch.Tensor:
  1003→        return _nvfp4_fake_quant_impl(x, block_size)
  1004→
  1005→    @staticmethod
  1006→    def backward(ctx, grad_output: torch.Tensor):
  1007→        return grad_output, None  # 纯 STE：梯度直通
  1008→
  1009→
  1010→def nvfp4_fake_quant(x: torch.Tensor, block_size: int = 16) -> torch.Tensor:
  1011→    """对张量做 NVFP4 伪量化（STE 梯度直通）。"""
  1012→    return NvFP4FakeQuant.apply(x, block_size)
  1013→
  1014→
  1015→# === 修改: arch/ffn.py — 在 FFN 中插入伪量化 ===
  1016→# class FeedForwardNetwork(nn.Module):
  1017→#     def __init__(self, embed_dim, ffn_dim, nvfp4_mode=0):
  1018→#         self.nvfp4_mode = nvfp4_mode
  1019→#         self.up_proj = MixPrecisionLinear(embed_dim, ffn_dim, bias=False)
